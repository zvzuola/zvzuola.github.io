ReactCSSTransitionGroup使用

详情看官网文档https://reactjs.org/docs/animation.html

You’ll notice that animation durations need to be specified in both the CSS and the render method; this tells React when to remove the animation classes from the element and — if it’s leaving — when to remove the element from the DOM.

在css和render方法里都必须指定动画时长，这是为了告诉react什么时候去移除动画class，或者什么时候去移除dom节点（如果是离开状态）

自定义动画类

transitionName可以传字符串，组件会默认加上-enter或-enter-active等，或者自己指定对象

初始挂载时的动画

During the initial mount ReactCSSTransitionGroup will get the example-appear CSS class and the example-appear-active CSS class added in the next tick.

在ReactCSSTransitionGroup组件自身挂载时会加上appear和appear-active

At the initial mount, all children of the ReactCSSTransitionGroup will appear but not enter. However, all children later added to an existing ReactCSSTransitionGroup will enter but not appear.

在组件初始化阶段，所有的子元素都会触发appear不会触发enter,所有在挂载之后加进来的子元素只会触发enter不会触发appear

划重点：就是在ReactCSSTransitionGroup组件自身挂载到dom节点是appear，当在ReactCSSTransitionGroup组件挂载之后，子元素的增加才会触发enter，所以要想enter能够生效，必须先初始化时挂载好ReactCSSTransitionGroup组件，比如下面的组件是不会看到enter效果的，因为ReactCSSTransitionGroup组件是动态创建出来的，此时只会触发appear，而transitionAppear默认是false
